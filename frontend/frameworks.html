Frameworks

We have covered the basics now. Here is a little deep dive into how most of web development looks like at the time of writing this.

I won’t go into much detail here, since it is basically a black hole where there is no return from, but

Many websites use frameworks and external libraries.

These basically mean javascript code that someone else has written

Frameworks are ways to write websites in a way that helps with things that would require a lot javascript taht is quite similar between different applications.

Collections of tested code that does the same thing so you don't need to write and figure out everything by yourself

It simplifies the way you write code, but can be hard to set up and you have to learn new ways to do things.

The idea with most frontend frameworks is that they follow what happens on the website and automatically update the associated values. Instead of manually having to tell that every time this button is pushed you need to update the 2nd, 3rd , 4rth etc, values individually, but it does it automatically.

For example some frameworks/libraries for frontend development are React, Vue, Angular, Svelte.
